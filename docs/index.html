Teach Claude to write grants with you

GrantAgent is a suite of 24 Claude skills covering the full lifecycle of a federal research grant — NIH, NSF, DoD, and foundations. From Specific Aims to the final format check.

Install in 5 minutes View on GitHub

How it works

A skill is a set of expert instructions Claude loads when it recognizes you need it. You never call skills by name — you describe what you're doing, and Claude brings in the right one.

1

Install once

Add the plugin in the Claude desktop app. No coding, nothing installed on your computer beyond Claude itself.

2

Describe your task

"I'm starting an NIH R01" or "my Research Strategy is over the page limit" — plain language is enough.

3

Work the proposal

Claude sets up an organized project folder, drafts section by section with your approval, and runs reviewer-style checks.

Install

Takes about five minutes in the Claude desktop app. You don't need a GitHub account.

  1. Open the plugins settings

    In the Claude desktop app, go to Settings → Customize → Plugins. (Menu wording varies slightly by version — look for "Plugins," "Capabilities," or "Extensions.")

  2. Add the marketplace

    Click Add Marketplace and enter exactly:

    joshua-d-campbell/GrantAgent
  3. Install the plugin

    In the grant-agent marketplace, install the grant-writing plugin. One plugin contains all 24 skills.

  4. Start writing

    Open a conversation and say, for example: "I'm starting a new NIH R01 application. Can you help me set up?"

Using Claude Code instead?

Two commands:

/plugin marketplace add joshua-d-campbell/GrantAgent
/plugin install grant-writing@grant-agent

Updates arrive automatically when the marketplace refreshes (/plugin marketplace update to force it).

The workflow

Skills follow the natural order of a proposal: setup, then the narrative and support documents in parallel, then review and polish. Solid arrows are the default sequence; dashed arrows are loops and feedback. Order is a default, not a rule.

The 24 skills

Every skill reads a shared project config, follows a strict versioning schema, never silently edits a document you haven't seen, and writes in your voice using a style profile built from your prior grants.

Foundation

grant-setup
Folder structure, FOA ingestion, checklist, config, style profile
grant-specific-aims
Aims page drafting and innovation feedback
grant-title
Title development after aims stabilize

Narrative

grant-significance
Significance section
grant-innovation
Innovation section
grant-approach
Approach, one subsection at a time, rationales first
grant-approach-experiment
Deep design review of a single experiment: controls, power, confounds, interpretation
grant-approach-math
Substance review of a single mathematical component: formulation, assumptions, identifiability, proof feasibility
grant-timeline-milestones
Timeline, milestones, Gantt
grant-abstracts
Summary, narrative, and lay abstracts

Support documents (parallel, any order)

grant-budget-justification
Personnel, line items, justification; syncs with aims
grant-letters-of-support
Early letter drafts for collaborators
grant-biosketch-support
Biosketches, Other Support / Current & Pending
grant-facilities-resources
Facilities, equipment, environment documents
grant-data-management-plan
NIH DMSP / NSF DMP
grant-compliance-sections
Human subjects, vertebrate animals, rigor

Review & polish

grant-mock-review
Red-team study section simulation
grant-math-notation
Math/LaTeX notation audit for math-heavy proposals
grant-condense
Cut an over-length draft to the limit: structural cuts, then sentence tightening
grant-proofread-structure
High-level flow, missing elements, transitions
grant-proofread-detail
Spelling, grammar, figure/table references
grant-references
Citation completeness and accuracy
grant-format-check
Page limits, fonts, margins, attachments

After review

grant-resubmission
Response to reviews and resubmission introduction

Things you can say

No commands to memorize — describe the stage you're at.

I'm starting a new NIH R01 application. Can you help me set up?
Review this experiment in my Approach — are the controls and sample size adequate?
My Research Strategy is over the 12-page limit. Help me cut it down.
Act as an NIH study section and critique my draft.
Check that every claim in my Significance section is properly cited.
Draft a letter of support for my collaborator to sign.
